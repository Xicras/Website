Positional vs Keyword arguements:
Keyword - func(a, b, c) where 'a', 'b', 'c' is used to define specific arguments. i.e., in command-> func(a=3) will update 'a' to 3 in the functional definition.
Position - func(a,b,c) where the position is used to define specific arguments i.e., in command -> func(3) will update 'a' to 3 as it is the first argument linked to the first position in the functional definition.

Lists:
Standard format:: New_list = [newitem for olditem in oldlist if <test>].
If struggling, try to repeat the exact same list in the new list (aka 'newitem' for 'olditem' in list will copy the same item from the old list to the new list) then think about the required functions to change the olditem into the new item (aka <test>), multiple operators can be added (and, or, etc).
Multiple lists can be checked by using the 'and' operator: new_list = [newitem for olditem in oldlist and oldlist2]

Dictionaries: Use "{}" instead of "[]".
Simplified formula: "new_dict_name = {newkey:newvalue for (oldkey, oldvalue) in dictionary.items() if test"
DO NOT FORGET '.items()'

Errors and Exceptions:
try: line(s) of code that is expected to cause an issue
except: do this (to fix) if there was an issue
else: do this if there was no issue
finally: do this after everything, no matter if there was or wasn't an issue
raise: prints string as message

File pathing:
Absolute pathing contains the entire path from the drive -> "C:\Users\eyuzo\Downloads\python-3.12.4-amd64.exe".
However, opening a file located in another folder requires pathing from the content root -> "..\.\python-3.12.4-amd64.exe"
".." is the same as "in parent folder" // "." is the same as "in the same folder"

Environmental Variables
When creating programs that interact with other people's systems, passwords and key-data will be required.
So, when creating authentication methods or logging into an API. Passwords can be saved within environmental variables (using the 'os' module) and saved locally instead of adding them as constants within the script itself.
CODE: import 'os'. VARIABLE = os.environ["VARIABLE_NAME"]. NOTE: The 'os' module can also be used to fully manage your drive!
If the variable does not exist, os.environ["VARIABLE_NAME"] = VARIABLE will instead create the variable which can be called later.
